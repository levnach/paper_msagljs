\begin{abstract}
We present an efficient edge routing algorithm that produces the shortest path in the homotopy class of an initial polyline route. The key idea is \emph{routing inside a channel}: given an initial path routed on a spanner graph, we construct a polygon (the channel) around the path using a Constrained Delaunay Triangulation, then apply the funnel algorithm to find the optimal path within that channel. Unlike prior approaches, our method does not require the channel polygon to be simple. We implement the algorithm in two open-source graph layout libraries, MSAGLJS (TypeScript) and MSAGL (C\#), and demonstrate its application to both force-directed and Sugiyama (layered) layouts. Experimental results show the method produces visually superior paths with competitive performance.
\end{abstract}

\section{Introduction}
\label{sec:intro}

A common approach to edge routing in graph drawing is to find shortest paths on a visibility graph that avoids node obstacles. However, the full visibility graph can have $O(n^2)$ edges, making this expensive for large graphs. A practical alternative, introduced by Dwyer and Nachmanson~\cite{dwyer2010fast}, routes edges on a spanner graph---a sparse approximation of the visibility graph---and then applies local optimizations to shorten the resulting paths.

We observe that the local shortcutting heuristic used in~\cite{dwyer2010fast} can fail to remove all kinks from a path. We replace it with a more principled optimization: for each edge, we construct a \emph{channel}---a polygon surrounding the initial path---and apply the funnel algorithm~\cite{chazelle1982theorem,hershberger1994computing} to find the shortest path within the channel's homotopy class.

The funnel algorithm has been used for edge routing before: Graphviz applies it in the Sugiyama layout where a simple containing polygon is readily available~\cite{dobkin1997implementing}. Our contribution is showing how to construct the channel for \emph{any} layout and demonstrating that the funnel algorithm works correctly even when the channel polygon is not simple.

We implement this approach in two open-source libraries: MSAGLJS\footnote{\url{https://github.com/microsoft/msagljs}} (TypeScript) and MSAGL\footnote{\url{https://github.com/microsoft/automatic-graph-layout}} (C\#), applying it to both force-directed and Sugiyama layouts.

\section{Related Work}
\label{sec:related}

Graphviz~\cite{graphviz} builds the full visibility graph for edge routing in force-directed layouts, which can be slow for large graphs. For Sugiyama layouts, it uses the funnel algorithm~\cite{dobkin1997implementing} but notes the requirement for a simple polygon. yWorks~\cite{yworks} offers ``Organic edge routing'' based on force-directed simulation, running in $O((n+m)\log(n+m))$ time. The approach of~\cite{dwyer2010fast} uses a Yao-graph spanner with local shortcutting, which we improve upon.

\section{Edge Routing Inside a Channel}
\label{sec:routing}

\subsection{Building the Spanner}

The edge routing starts by building a spanner graph, an approximation of the full visibility graph, following~\cite{dwyer2010fast}. The spanner is built on a variation of a Yao graph~\cite{flinchbaugh1981strong,yao1982constructing}. Each node is covered by a polygon with at most $8$ corners, which serve as vertices of the spanner. For each vertex, a family of $k$ cones (with angle $\frac{2\pi}{k}$, $k=12$ in our setting) partitions the plane; for each cone, at most one edge is created to the closest vertex inside it. The resulting spanner has $O(N)$ edges, where $N$ is the number of graph nodes.

\subsection{Channel Construction}
\label{sec:channel}

Before routing edges, we compute a Constrained Delaunay Triangulation (CDT) \cdt{} on the set of obstacle polygons \obst{}~\cite{delaunay1934sphere,domiter2008sweep}.

For each graph edge, we route an initial path \unpath{} on the spanner. Let $\mathcal{S}$ and $\mathcal{E}$ be the obstacles containing the start and end points of \unpath. We define \triset, the set of all triangles $t \in \mathcal{T}$ such that either $t \subset \mathcal{S} \cup \mathcal{E}$, or $t$ intersects \unpath{} and is not inside any obstacle in $\mathcal{O} \setminus \{\mathcal{S}, \mathcal{E}\}$. The union of \triset{} forms polygon \plg---the \emph{channel}. The boundary of \plg{} consists of all triangle sides that belong to exactly one triangle in \triset.

\subsection{Sleeve and Funnel Optimization}

To apply the funnel algorithm, we need a \emph{sleeve}: a sequence of triangles from start to end where consecutive triangles share a side. Since \triset{} may not have this property, we compute a new Constrained Delaunay Triangulation of \plg{} with the boundary edges as constraints, trace \unpath{} through it, and extract the sleeve. The funnel algorithm then produces the shortest path in the homotopy class of \unpath.

Crucially, \plg{} need not be simple. The funnel algorithm requires only a valid sleeve, not a simple polygon. When \plg{} self-intersects, the resulting path is the shortest within the sleeve's homotopy class, though not necessarily the globally shortest path inside \plg{} (see Fig.~\ref{fig:non_optimal_path}).

\subsection{Application to Sugiyama Layout}

The channel-based routing naturally extends to Sugiyama (layered) layouts. In a Sugiyama layout, edges already follow polyline paths through the layers. These polylines serve as the initial paths \unpath, and the same CDT-based channel construction and funnel optimization apply. This generalizes the Graphviz approach~\cite{dobkin1997implementing}, which requires a simple containing polygon that is available only in specific configurations.

\section{Direct Routing on the CDT}
\label{sec:corridor}

The channel-based approach of Section~\ref{sec:routing} requires two steps: an initial route on the spanner graph, followed by channel construction and funnel optimization. We now describe an alternative that eliminates the spanner entirely, routing edges directly on the CDT dual graph.

\subsection{A* Search on the Dual Graph}

Given the CDT \cdt{} built on padded obstacle polylines, we define a \emph{dual graph} whose nodes are the CDT triangles and whose edges connect triangles sharing a side. For each graph edge from source~$s$ to target~$t$, we locate the triangle containing~$s$ and run A*~\cite{hart1968formal} on this dual graph to find a shortest sequence of triangles---the \emph{sleeve}---leading to the triangle containing~$t$.

The A* edge weight between adjacent triangles is the Euclidean distance between their centroids, and the heuristic is the straight-line distance from the current centroid to~$t$. A triangle is excluded from the search if all three of its CDT sites belong to the same obstacle (indicating the triangle lies inside an obstacle interior). Triangles inside the source or target obstacles are permitted, since the path must traverse those obstacles.

The sleeve produced by A* is then passed to the funnel algorithm~\cite{chazelle1982theorem}, which computes the shortest path through the sleeve's sequence of diagonals (see Fig.~\ref{fig:cdt_dual}).

\subsection{Vertex Collapse at Source and Target}
\label{sec:collapse}

A subtlety arises at the source and target nodes. The sleeve passes through triangles whose vertices lie on the padded obstacle boundaries of the source node~$S$ and target node~$T$. When these boundary vertices appear as diagonal endpoints, the funnel algorithm threads the path through narrow gaps between obstacle corners, producing sharp zigzag turns near each endpoint (Fig.~\ref{fig:channel_collapse}a).

We eliminate these artifacts with a \emph{vertex collapse} operation applied when converting the sleeve into funnel diagonals. The sleeve is a sequence of triangles $t_1, t_2, \ldots, t_k$. Each consecutive pair $(t_i, t_{i+1})$ shares an edge---the $i$-th \emph{diagonal} $d_i$---with two CDT vertex endpoints. In the CDT, each vertex records the obstacle polyline it belongs to (its \emph{owner}).

The collapse proceeds as follows. For each diagonal $d_i$ with vertex endpoints $u$ and $v$:
\begin{enumerate}
  \item If $u$'s owner is the source obstacle polyline, replace $u$'s position with the source center~$s$. Similarly, if $v$'s owner is the target obstacle polyline, replace $v$'s position with the target center~$t$.
  \item If both endpoints collapse to the same point (e.g., both belong to the source obstacle), the diagonal becomes degenerate and is removed from the sequence.
  \item The left/right orientation of each diagonal is determined from the \emph{original} (uncollapsed) geometry, preserving the topological structure of the sleeve.
\end{enumerate}

The effect on the funnel is illustrated in Fig.~\ref{fig:channel_collapse}b. Before collapse, the diagonals near $S$ fan out from distinct obstacle corner vertices $a_1, a_2, \ldots$, forcing the funnel through narrow gaps. After collapse, all these endpoints merge to the single point~$s$, so the funnel sees a wide opening and naturally selects the optimal exit direction. The same occurs at the target end, where corners $b_1, b_2, \ldots$ collapse to~$t$. Diagonals whose both endpoints belonged to the same obstacle (e.g., an edge of the source obstacle's padded boundary) become degenerate and are removed---they would have contributed no useful constraint to the funnel.

A final arrowhead-trimming step clips the resulting path at the actual node boundary curves, so the visible edge begins and ends cleanly at each node's perimeter.

\subsection{Comparison with the Channel Approach}

The direct CDT routing avoids building a spanner graph entirely, reducing both code complexity and runtime for the initial routing step. The A* search on the dual graph takes $O(k \log k)$ time per edge, where $k$ is the number of triangles visited. Since the CDT has $O(N)$ triangles for $N$ obstacle vertices, the total routing cost is $O(m \cdot N \log N)$ in the worst case for $m$~edges, though in practice the A* heuristic keeps the search local.

We measured the total edge length produced by each method on the same node layout. On two benchmark graphs---a 407-node social network (2\,639 edges) and a 1\,463-node graph from the Graphviz test suite---the corridor method produces paths only 2.7--2.8\% longer than the spanner-based method, while being significantly simpler to implement.

\section{Conclusion}
\label{sec:conclusion}

We presented an edge routing method based on constructing a channel around an initial path and applying the funnel algorithm to find the shortest path in its homotopy class. The method does not require the channel polygon to be simple, making it applicable to any graph layout. We demonstrated its effectiveness in both force-directed and Sugiyama layouts, implemented in the open-source MSAGLJS and MSAGL libraries.
